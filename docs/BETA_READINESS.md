# Mealandmotion Beta Readiness

## Beta scope
- Personal Trainers are the primary customer during beta.
- Personal Trainers operate primarily on web.
- Clients use mobile first, with web as a backup.
- Managed clients follow trainer-assigned plans rather than generating independent programmes.

## Beta promise
- A Personal Trainer can invite a client, activate them, assign plans, message them, and request feedback.
- A client can accept the invite, understand the relationship, view plans, log adherence, and message the trainer.

## Go / no-go checklist
- Public Privacy Policy exists
- Public Terms of Service exists
- Public Health Disclaimer exists
- Public FAQ exists
- Public Support page exists
- PT dashboard and roster are usable
- Managed-client restrictions are enforced
- Client post-acceptance state is clear
- Bug reporting path exists on web and mobile
- Fresh-account QA pass completed

## Beta support process
- Support email: support@mealandmotion.app
- Urgent issues: login, invite acceptance, missing plans, trainer/client linkage
- All tester bugs should include role, platform, expected result, actual result, and screenshot if possible

## Known beta positioning
- AI is a differentiator, not the core promise
- This is a coaching operating system, not a scheduling or marketplace platform
- Multi-trainer clinic workflows are out of scope for the first beta
